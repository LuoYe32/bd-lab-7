# Модель кластеризации на PySpark

## Описание работы

В рамках работы был использован датасет OpenFoodFacts, содержащий информацию о пищевой ценности продуктов. На основе числовых признаков продуктов была реализована модель кластеризации, позволяющая разбить продукты на группы со схожими характеристиками.

## Датасет

Использовался датасет OpenFoodFacts:

https://world.openfoodfacts.org/data

Полный датасет имеет большой размер (более 13 ГБ в архиве), поэтому была выполнена предварительная обработка данных и ограничение выборки в соответствии с ресурсами локального компьютера.

Для кластеризации использовались следующие признаки:

- energy_100g
- fat_100g
- carbohydrates_100g
- sugars_100g
- proteins_100g

Дополнительно выполнялась очистка данных:
- удаление пропусков
- удаление отрицательных значений
- удаление выбросов
- ограничение размера выборки

## Настройка Spark

Spark запускался локально через PySpark.

Использовались следующие параметры конфигурации:

- `master = local[*]` — использование всех ядер процессора
- `spark.driver.memory = 4g`
- `spark.executor.memory = 4g`
- `spark.sql.shuffle.partitions = 8`
- `spark.serializer = KryoSerializer`
- `spark.sql.adaptive.enabled = true`
- `spark.sql.autoBroadcastJoinThreshold = 10MB`
- `spark.sql.inMemoryColumnarStorage.compressed = true`

Данные настройки позволили оптимизировать выполнение приложения на локальной машине.

## Этапы pipeline

### 1. Загрузка данных
Чтение CSV файла через Spark DataFrame API.

### 2. Предобработка данных
Очистка данных от пропусков, отрицательных значений и выбросов.

### 3. Построение признаков
Использование:
- VectorAssembler
- StandardScaler

### 4. Обучение модели
Использовался алгоритм K-Means.

Параметры модели:
- k = 5
- seed = 51

### 5. Оценка качества
Использовалась метрика Silhouette Score.

Итоговый результат:

**Silhouette Score = 0.7189**

Также выводились:
- размеры кластеров
- центры кластеров
